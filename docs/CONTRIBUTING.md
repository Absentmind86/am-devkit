# Contributing

Read [`PROJECT.md`](PROJECT.md) for scope and layer rules. Pre-release: prefer small, focused changes aligned with the current phase in `AGENTS.md`. Before a tagged release, use the VM checklist in [`RELEASE_TESTING.md`](RELEASE_TESTING.md).

If you add a **direct** Python dependency in `requirements.txt` or integrate a new **external** tool, update [`THIRD_PARTY_NOTICES.md`](THIRD_PARTY_NOTICES.md) (license name, link, and a one-line description of use).
